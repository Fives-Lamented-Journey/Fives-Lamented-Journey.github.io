一、最根本前提：门 = x86 CPU 硬件机制

**门不是操作系统实现的，是 CPU 原生支持的安全跳转机制。**

- 门是一个 **8 字节固定格式的数据结构**
- 存在于 **GDT / LDT / IDT** 这些描述符表中
- 作用：**允许低特权级代码（用户态）安全进入高特权级（内核态）**
- 全程由 CPU 硬件自动完成：权限检查 → 栈切换 → 跳转 → 返回

------

# 二、CPU 如何识别“这是一个门”？

CPU 不看名字、不看位置，**只看 TYPE 字段**。

门描述符通用格式（8 字节）：

```
63            48 47 46 45 44 43 42 40 39           32
┌───────────────┬───┬─────┬───────┬────────────────┐
│ OFFSET 31~16  │ P │ DPL │  TYPE │   SELECTOR     │
├───────────────┴───┴─────┴───────┴────────────────┤
│              OFFSET 15~0                         │
└──────────────────────────────────────────────────┘

```

关键字段：

- **P=1**：描述符有效
- **DPL**：描述符特权级（允许什么权限访问）
- **TYPE（4 位）**：**CPU 唯一识别依据**

TYPE 与门类型对应：

- `0100` → 任务门 Task Gate
- `0101` → 调用门 Call Gate
- `0110` → 中断门 Interrupt Gate
- `0111` → 陷阱门 Trap Gate

**只要 TYPE 是这四个之一，CPU 就按“门”来处理。**

------

# 三、内核到底是怎么“实现门”的？

内核不创造门，内核只是**按 CPU 格式填表**。

## 步骤 1：准备高特权代码

内核先写好一段 **Ring0 代码**：

- 中断处理函数
- 系统调用入口
- 异常处理函数

## 步骤 2：构造 8 字节门描述符

内核要填写：

1. 目标代码段选择子（必须是内核代码段）
2. 目标函数的线性地址偏移
3. DPL（控制谁能访问：用户态访问设为 3）
4. TYPE（决定这是什么门）
5. P=1

## 步骤 3：把描述符放到指定的表中

必须遵守 **CPU 硬件位置限制**（不是规范，是强制）。

## 步骤 4：加载表基址

- `LGDT` → 加载 GDT
- `LIDT` → 加载 IDT

之后 CPU 就认这些门了。

------

# 四、门到底能放在哪张表？（CPU 死规定）

这是你最关键的疑问，我给你**最精确、最底层**的规则：

## 1. 中断门、陷阱门

- **只能放在 IDT**
- 不能放在 GDT/LDT
- 原因：`INT n` / 硬件中断 **只访问 IDT**

## 2. 调用门

- **只能放在 GDT 或 LDT**
- 不能放在 IDT
- 原因：`CALL/JMP 远调用` 只查 GDT/LDT

## 3. 任务门

- **GDT / LDT / IDT 都可以放**
- 唯一真正通用的门

------

# 五、为什么不能乱放？（硬件通路决定）

CPU 从设计上就分了**两条完全独立的查表通路**：

## 通路 1：中断/异常/INT n

→ 直接索引 **IDT**

→ 只能识别：中断门、陷阱门、任务门

## 通路 2：远调用 / 远跳转 CALL/JMP cs:offset

→ 索引 **GDT/LDT**

→ 只能识别：调用门、任务门、段描述符

**位置不是规范，是硬件物理限制。**

------

# 六、门如何实现低特权 → 高特权？（硬件自动流程）

所有门的共同逻辑，CPU 自动做：

## 1. 特权检查

- 当前 CPL ≤ 门 DPL（你有权进这扇门）
- 当前 CPL ≥ 目标代码段 DPL（只能从低往高跳）

不满足 → #GP 异常

## 2. 栈切换（跨特权时必须）

- 从 **TSS（任务状态段）** 中取出高特权级栈：SS0:ESP0
- 把当前栈的关键信息压入新栈
- 防止用户栈篡改、内核栈污染

## 3. 保存上下文

- 压入寄存器、返回地址
- 中断门会自动 **关中断（清 IF）**
- 陷阱门不关闭中断

## 4. 跳转高特权代码

- CS 加载目标代码段
- EIP 加载目标偏移
- CPL 自动变成高特权级（如 0）

## 5. 返回

- 调用门 → `RETF`
- 中断/陷阱门 → `IRET`
- 恢复上下文、恢复栈、恢复 CPL

------

# 七、四类门详细对比（最完整版）

## 1. 调用门 Call Gate（TYPE=0101）

- 位置：GDT/LDT
- 触发：`CALL FAR 选择子:偏移`
- 作用：**主动跨特权级函数调用**
- 特点：
  - 硬件自动复制参数（最多 31 个）
  - 可用户→内核
  - 返回用 `RETF`

## 2. 中断门 Interrupt Gate（TYPE=0110）

- 位置：IDT
- 触发：硬件中断、`INT n`
- 作用：处理中断
- 特点：
  - **进入自动关中断 IF=0**
  - 返回用 `IRET`，自动恢复 IF

## 3. 陷阱门 Trap Gate（TYPE=0111）

- 位置：IDT
- 触发：异常、断点
- 作用：处理异常、调试
- 特点：
  - **不关闭中断**
  - 可嵌套、可重入

## 4. 任务门 Task Gate（TYPE=0100）

- 位置：GDT/LDT/IDT
- 触发：`CALL/JMP` 或中断
- 作用：**任务切换**
- 特点：
  - 切换整个 TSS
  - 自动保存/恢复全部寄存器

------

# 八、最核心结论

1. **门是 CPU 硬件定义的 8 字节描述符，不是软件抽象。**
2. **CPU 只通过 TYPE 识别门，与位置无关。**
3. **门能放哪张表，是 CPU 硬件强制限制，不是规范。**
4. **内核只负责按格式填写门，并放到正确的表中。**
5. 跨特权级跳转由 **CPU 自动完成**：权限检查 → 栈切换 → 跳转 → 返回。
6. 所有门的目的：**让低特权代码安全、受控地进入高特权代码。**
